# 知识库文档格式规范 v1.0

> 数据组产出、pipeline 组消费的唯一接口。所有人严格遵守，互不阻塞。

## 1. 存放位置与命名

```
knowledge_base/
├── 政策/
│   ├── 转专业政策_教务处_2025.md
│   └── 保研推免细则_计算机学院_2025.md
├── 通知/
│   └── 2026春季奖学金申报通知.md
└── FAQ/
    └── 宿舍常见问题.md
```

- 一律 UTF-8 编码的 `.md` 文件
- 文件名：`主题_来源单位_年份.md`（通知类可用日期开头）
- 一个文件 = 一个独立主题，别把多个政策塞一个文件

## 2. 文件结构：frontmatter + 正文

每个文件开头必须有 YAML frontmatter：

```markdown
---
title: 本科生转专业管理办法
source_url: https://jwc.xxx.edu.cn/info/1023/4567.htm
source_org: 教务处
publish_date: 2025-09-01
category: 政策            # 政策 | 通知 | FAQ
tags: [转专业, 教务]
valid: true               # 已失效的旧文档改为 false
---

# 本科生转专业管理办法

## 申请条件
...正文...
```

## 3. 字段说明

| 字段 | 必填 | 说明 |
|---|---|---|
| title | 必填 | 官方标题 |
| source_url | 必填 | 原文链接，**回答溯源全靠它，绝不能省** |
| source_org | 必填 | 发布单位（教务处/学工部/XX学院） |
| publish_date | 必填 | 发布日期 YYYY-MM-DD，未知则填抓取日期并加 tag `日期存疑` |
| category | 必填 | 三选一：政策 / 通知 / FAQ |
| tags | 可选 | 2~5 个关键词，方便过滤 |
| valid | 可选 | 默认 true |

## 4. 正文清洗规则

1. 保留原文的标题层级（`##`、`###`）——RAG 按标题切块，层级就是切块边界
2. 表格转成 markdown 表格，不要截图
3. 删除页眉页脚、"点击量"、导航栏等网页噪音
4. 附件（如申请表）不粘贴内容，写一行：`附件：XX申请表，见原文链接`
5. 不要自己改写/总结原文——原文怎么写就怎么留，解读是 LLM 的事

## 5. 验收标准

- [ ] frontmatter 五个必填字段齐全，YAML 能被解析
- [ ] source_url 点开能访问
- [ ] 正文无乱码、无网页噪音
- [ ] 单文件不超过 ~8000 字，超了按章节拆成多个文件
